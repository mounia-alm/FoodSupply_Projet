# Guide d'evaluation - Projet Livrili

## Configuration requise

- Docker Desktop installe
- Git installe
- Navigateur web (Chrome, Firefox ou Edge)

---

## Etape 1 : Cloner le projet

git clone https://github.com/nom-utilisateur/Livrili-Microservices.git
cd Livrili-Microservices

---

## Etape 2 : Demarrer Docker Desktop

Ouvrez Docker Desktop et attendez que le moteur soit en marche (icone verte).

---

## Etape 3 : Lancer les services

docker-compose up -d

Attendez 30 secondes que tous les conteneurs soient demarres.

---

## Etape 4 : Verifier l'etat des conteneurs

docker ps --format "table {{.Names}}\t{{.Ports}}\t{{.Status}}"

Resultat attendu : 8 conteneurs avec le statut "Up"

- auth-service : port 8001
- user-service : port 8002
- product-service : port 8003
- order-service : port 8004
- consul : port 8500
- rabbitmq : port 15672
- traefik : ports 80 et 8080
- frontend : port 5173

---

## Etape 5 : Tester les URLs

Frontend : http://localhost:5173
API Produits : http://localhost:8003/api/products/
API Auth : http://localhost:8001/api/auth/login/
Consul : http://localhost:8500
RabbitMQ : http://localhost:15672 (login: guest, mot de passe: guest)
Traefik : http://localhost:8080

---

## Etape 6 : Se connecter au frontend

1. Ouvrez http://localhost:5173
2. Email : admin@example.com
3. Mot de passe : admin123

---

## Etape 7 : Tester les fonctionnalites

Cote Restaurateur (theme orange) :

- Consulter la liste des produits
- Filtrer les produits par fournisseur
- Rechercher un produit
- Ajouter un produit au panier
- Modifier la quantite dans le panier
- Passer une commande
- Annuler une commande

Cote Fournisseur (theme vert) :

- Consulter la liste des produits
- Ajouter un nouveau produit
- Modifier un produit existant
- Supprimer un produit
- Voir les commandes recues

---

## Etape 8 : Tester la validation de stock

1. Connectez-vous en tant que restaurateur
2. Choisissez un produit avec un stock faible
3. Commandez une quantite superieure au stock
4. Message attendu : "Requested quantity exceeds available stock"

Cette validation est faite cote backend pour eviter les contournements.

---

## Etape 9 : Preuves techniques

Preuve 1 : Conteneurs Docker
docker ps --format "table {{.Names}}\t{{.Ports}}"

Preuve 2 : Adresses IP distinctes
docker network inspect foodsupply_projet_default | findstr "Name\" \"IPv4Address"

Preuve 3 : API REST
curl http://localhost:8003/api/products/

Preuve 4 : Consul (Service Registry)
Ouvrez http://localhost:8500 et cliquez sur "Services"

Preuve 5 : Traefik (Reverse Proxy)
Ouvrez http://localhost:8080 et cliquez sur "HTTP Routers"

Preuve 6 : RabbitMQ (Asynchrone)
Ouvrez http://localhost:15672 (guest/guest) et cliquez sur "Queues"

---

## Etape 10 : Arreter les services

docker-compose down

---

## Grille d'evaluation

Conception : 25 points
Implementation : 30 points
Deploiement multi-serveurs : 20 points
Asynchrone + securite : 15 points
Presentation + documentation : 10 points
Total : 100 points

---

## Problemes courants

Port deja utilise : netstat -ano | findstr :8001
Docker ne demarre pas : Redemarrez Docker Desktop
Container en erreur : docker-compose logs nom_container
Page blanche : Verifiez que tous les services sont "Up"
